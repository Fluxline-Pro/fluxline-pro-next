**INTELLECTUAL PROPERTY DISCLAIMER**: This project is the proprietary intellectual property of Intermountain Health. ## Project Overview

Architected and delivered **MyIntermountain**, a scalable enterprise intranet platform for Intermountain Health built on SharePoint Online with comprehensive Azure backend services, automated provisioning workflows, and enterprise SSO integration. Delivered complete infrastructure within **9 months** as Scrum lead coordinating cross-functional teams.

**Project Scope:**

- **Azure backend architecture**: Authentication, provisioning, and automation services
- **SharePoint Online frontend**: Microsoft 365-integrated employee experience
- **Automated provisioning**: Power Automate and Spark-based workflows
- **Enterprise SSO**: Azure Entra ID and PING integration
- **Timeline**: 9 months from architecture to production delivery
- **Leadership**: Scrum lead managing agile delivery, QA, and CI/CD pipeline

## Technical Implementation

### Azure Backend Architecture

**Infrastructure Services:**

- Azure Functions for serverless backend logic
- Power Automate for workflow automation and provisioning
- Azure Entra ID (formerly Azure AD) for identity management
- Microsoft Graph API for M365 service integration
- Automated content and permissions management

**Authentication & Identity:**

- Azure Entra ID single sign-on (SSO) integration
- PING SSO for enterprise authentication
- Role-based access control (RBAC) and permissions management
- Seamless OneDrive and SharePoint Online authentication

### SharePoint Online & Microsoft 365 Integration

**Frontend Experience:**

- SharePoint Online as primary employee portal
- Microsoft 365 service integration (Teams, OneDrive, Planner)
- SPFx (SharePoint Framework) web parts for custom functionality
- Microsoft Search integration for unified content discovery
- Responsive design for desktop and mobile access

**Content Management:**

- OneDrive integration for personal file storage
- SharePoint Online document libraries and lists
- Permissions management via Graph API
- Content migration and organization workflows

### Provisioning & Automation

**Spark Provisioning Tool:**

- Custom provisioning application built with C#
- Automated site creation and configuration
- Template-based provisioning workflows
- Integration with Power Automate for approval processes

**Power Automate Workflows:**

- Automated provisioning request handling
- Content approval and governance workflows
- User onboarding and offboarding automation
- Integration with Azure backend services

## Challenges & Solutions

### Challenge: Complex Provisioning Requirements

**Solution**: Developed Spark provisioning tool with C# and integrated Power Automate workflows to handle diverse site creation needs, approval processes, and automated configuration.

### Challenge: Azure Entra ID & PING SSO Integration

**Solution**: Architected seamless authentication layer bridging Azure Entra ID with existing PING SSO infrastructure, ensuring single sign-on across all services.

### Challenge: 9-Month Delivery Timeline

**Solution**: Led agile delivery as Scrum lead, maintaining consistent sprint velocity, coordinating cross-functional dependencies, and establishing CI/CD automation for rapid iteration.

### Challenge: Microsoft 365 Service Integration

**Solution**: Leveraged Microsoft Graph API to unify SharePoint, OneDrive, Teams, and Microsoft Search into cohesive employee experience platform.
